NekoHTML 0.3.2

About

NekoHTML is a simple HTML scanner and tag balancer that enables application programmers to parse HTML documents and access the information using standard XML interfaces. The parser can scan HTML files and "fix up" many common mistakes that human (and computer) authors make in writing HTML documents. NekoHTML adds missing parent elements; automatically closes elements with optional end tags; and can handle mismatched inline element tags.

NekoHTML is written using the Xerces Native Interface (XNI) that is the foundation of the Xerces2 implementation. This enables you to use the NekoHTML parser with existing XNI tools without modification or rewriting code.

Usage Instructions

NekoHTML is designed to be as lightweight and simple to use as possible. Using the Xerces 2.0.0 parser as a foundation, NekoHTML is transparent for applications that instantiate parser objects with the Java API for XML Processing (JAXP). Just put the NekoHTML jar file in the classpath before the Xerces jar files. For example (on Windows): 

> java -cp nekohtml.jar;xmlParserAPIs.jar;xercesImpl.jar;xercesSamples.jar 
       sax.Counter doc/index.html
doc/index.html: 10 ms (69 elems, 35 attrs, 0 spaces, 3512 chars)

Alternatively, you can construct any XNI-based parser class using the HTMLConfiguration parser configuration class found in the org.cyberneko.html package. For example: 

import org.apache.xerces.parsers.AbstractSAXParser;
import org.cyberneko.html.HTMLConfiguration;

public class MySAXParser extends AbstractSAXParser {

    public MySAXParser() {
        super(new HTMLConfiguration());
    }

}

The above example shows how to write a SAX parser using Xerces2 and the HTMLConfiguration class. The same can be used to create a custom NekoHTML DOM parser. However, when doing so, it would be a good idea to set the implementation used by the DOM parser to construct the DOM tree so that an HTML DOM document is constructed. Using Xerces2, you can set the "http://apache.org/xml/properties/dom/document-class-name" property to the value "org.apache.html.dom.HTMLDocumentImpl". Refer to the Xerces documentation for setting parser properties.

Requirements and Limitations

This version of NekoHTML requires the following:

  • Java 1.1 (or higher)
  • Xerces 2.0.0

This version has the following limitations:

  • There are no major limitations with this release. However, additional work can always be done to improve performance, fix bugs, and add functionality.

More bug fixes to allow the parser to be used with Xalan 2.3.0. The parser wasn't keeping track of features and properties and without namespaces turned on, Xalan would not correctly transform the SAX events emitted using NekoHTML.

Minor bug fix to work around problem in Xerces 2.0.0 SAX parser that drops attributes when parser configuration doesn't have a symbol table.

Version 0.1 (04 Feb 2002)
Initial writing.
